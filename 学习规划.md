# STM32 + FreeRTOS 油烟机项目学习规划

## 目标

通过这个项目，逐步掌握一个基于 STM32F103C8T6、标准外设库和 FreeRTOS 的完整嵌入式应用：从工程结构、外设驱动、任务调度、传感器采集、电机控制，到 IAP 固件升级。

建议学习顺序是：先跑通工程，再读主流程，然后分模块拆解，最后自己改功能。

## 第 0 阶段：准备环境

目标：能打开、编译、下载、观察现象。

要做的事：

- 安装 Keil MDK-ARM，并确认能打开 `USER/project.uvprojx`。
- 确认芯片型号、开发板、下载器、串口工具和供电方式。
- 在 Keil 中完整 Rebuild 一次，确认没有错误。
- 下载 `OBJ/PWM.hex` 或重新编译生成的程序到板子。
- 记录按键、LCD、电机、传感器、串口的实际表现。

检查点：

- 能说明工程入口是 `USER/main.c`。
- 能说明硬件初始化在 `Hardware_Init()` 中完成。
- 能说明 FreeRTOS 调度器由 `vTaskStartScheduler()` 启动。

## 第 1 阶段：看懂工程结构

目标：知道每个文件夹负责什么。

阅读顺序：

1. `文件夹说明.txt`
2. `USER/main.c`
3. `USER/main.h`
4. `APP_TASK/app_tasks.h`
5. `APP_TASK/app_tasks.c`

重点理解：

- `USER`：Keil 工程、主函数、中断入口和 STM32 系统文件。
- `APP_TASK`：应用层任务、系统状态、模式切换逻辑。
- `BSP`：板级驱动，比如电机、按键、LCD、MQ2、DHT11、DMA、IAP。
- `SYSTEM`：串口、延时、系统辅助代码。
- `FreeRTOS`：RTOS 内核源码和配置。
- `STM32F10x_FWLib`：STM32 标准外设库。
- `tools`：固件 CRC 处理脚本和 bin 文件。

输出要求：

- 自己画一张工程结构图。
- 用自己的话解释从上电到创建任务的流程。

## 第 2 阶段：掌握 FreeRTOS 基础

目标：能看懂任务、优先级、信号量和延时。

重点文件：

- `FreeRTOS/include/FreeRTOSConfig.h`
- `APP_TASK/app_tasks.h`
- `APP_TASK/app_tasks.c`
- `SYSTEM/delay/delay.c`

重点概念：

- 任务创建：`xTaskCreate`
- 任务优先级：`TASK_*_PRIORITY`
- 任务栈大小：`TASK_*_STK_SIZE`
- 互斥量：`xSemaphoreCreateMutex`
- 二值信号量：`xSemaphoreCreateBinary`
- 中断释放信号量：`xSemaphoreGiveFromISR`
- 任务延时：`delay_ms` 和 `vTaskDelay`

建议实验：

- 修改 UI 任务刷新周期，比如从 200 ms 改成 500 ms，观察 LCD 刷新变化。
- 修改按键任务周期，比如从 10 ms 改成 20 ms，观察按键响应。
- 打开栈溢出检测前，先理解 `configCHECK_FOR_STACK_OVERFLOW` 的作用。

检查点：

- 能解释为什么 `TIM4_IRQHandler` 里不能直接做复杂计算。
- 能解释为什么速度计算任务通过信号量被中断唤醒。
- 能说明各任务的大致周期。

## 第 3 阶段：读懂系统状态机

目标：理解项目的核心业务逻辑。

重点文件：

- `APP_TASK/app_tasks.h`
- `APP_TASK/app_tasks.c`

重点结构：

- `SystemState_t`
- `WorkMode_t`
- `SpeedLevel_t`
- `AutoModeState_t`

重点函数：

- `System_Init`
- `System_SwitchMode`
- `System_SwitchSpeedLevel`
- `System_ToggleMotor`
- `MotorControlTask`
- `AntiBackflowTask`
- `UIDisplayTask`

需要搞清楚的模式：

- 待机模式：电机停止，但传感器和风速算法仍运行。
- 手动模式：按档位设定目标转速，由 PID 控制电机。
- 自动模式：根据温湿度和气体浓度判断 Cooking Event，再调节风速。
- 防回流模式：根据气体浓度阈值自动启停风机。

建议实验：

- 在 LCD 上增加当前 `targetRPM` 显示。
- 修改 `SPEED_LOW_RPM` 和 `SPEED_HIGH_RPM`，观察转速变化。
- 修改自动模式启动等待时间 `AUTO_MODE_STARTUP_TIME`，方便快速验证逻辑。

检查点：

- 能画出模式切换状态图。
- 能说明短按、长按分别影响哪些状态变量。

## 第 4 阶段：学习外设驱动

目标：掌握项目中主要外设的初始化和使用方法。

建议阅读顺序：

1. `BSP/GPIO/gpiox.c`
2. `BSP/KEY/key.c`
3. `BSP/BEEP/beep.c`
4. `SYSTEM/usart/usart.c`
5. `BSP/LCD/lcd.c`
6. `BSP/DHT11/dht11.c`
7. `BSP/MQ2/mq2.c`
8. `BSP/MOTOR/motor.c`

重点理解：

- GPIO 输入输出模式。
- 按键消抖和短按/长按事件。
- 串口 printf 重定向。
- LCD 字符显示流程。
- DHT11 单总线时序。
- MQ2 模拟量采集和换算。
- TIM1 互补 PWM 驱动电机。

建议实验：

- 给按键事件增加串口打印。
- 单独测试蜂鸣器短响和长响。
- 在 LCD 上显示温湿度和 MQ2 原始值。

检查点：

- 能说明每个外设用到了哪些 GPIO、定时器、DMA 或 USART。
- 能定位一个外设初始化函数和一个外设使用函数。

## 第 5 阶段：学习电机测速和 PID 控制

目标：理解闭环控制链路。

重点文件：

- `BSP/MOTOR/motor.c`
- `BSP/MOTOR/motor.h`
- `BSP/PID/pid.c`
- `BSP/PID/pid.h`
- `BSP/WIND/wind_speed.c`
- `BSP/WIND/wind_speed.h`

重点链路：

1. TIM2 编码器模式采集脉冲。
2. TIM4 周期中断通知 `SpeedCalcTask`。
3. `get_speed` 计算实际转速。
4. `MotorControlTask` 设置目标转速。
5. `PID_Calculate` 输出 PWM。
6. `motor_pwm_set` 更新 TIM1 占空比。

建议实验：

- 串口打印 `targetRPM`、`actualRPM`、PID 输出。
- 调整 PID 参数，观察响应速度和稳定性。
- 暂时关闭积分项，比较控制效果。

检查点：

- 能说明 PID 的输入、目标和输出分别是什么。
- 能说明编码器测速为什么需要滤波。

## 第 6 阶段：学习自动风速算法

目标：理解传感器融合到风速输出的算法。

重点文件：

- `BSP/WIND/wind_speed.h`
- `BSP/WIND/wind_speed.c`
- `APP_TASK/app_tasks.c`

重点参数：

- `TEMP_BASE`
- `TEMP_MAX`
- `HUMIDITY_BASE`
- `HUMIDITY_MAX`
- `GAS_BASE`
- `GAS_MAX`
- `WEIGHT_TEMP`
- `WEIGHT_HUMIDITY`

检查点：

- 能说明温度、湿度、气体浓度如何归一化。
- 能说明 PWM 百分比如何映射到定时器 CCR。

## 第 7 阶段：学习 IAP 和 CRC32

目标：理解串口 DMA 接收固件、CRC 校验、写 Flash、跳转 APP 的流程。

重点文件：

- `BSP/DMA/dma.c`
- `BSP/DMA/dma.h`
- `BSP/IAP/iap.c`
- `BSP/IAP/iap.h`
- `BSP/STMFLASH/stmflash.c`
- `BSP/CRC32/crc32.c`
- `tools/add_crc32.py`

重点流程：

1. 串口 DMA 接收固件。
2. DMA 传输完成中断释放 IAP 信号量。
3. `iap_task` 校验 CRC32。
4. 擦除 APP 区 Flash。
5. 写入新固件。
6. 设置 MSP 并跳转 APP。

必须注意：

- 当前 `FLASH_APP1_ADDR` 和 STM32F103C8 的 64 KB Flash 容量需要重新核对。
- Bootloader 和 APP 的 Flash 分区、RAM 分区不能重叠。
- `receive_buff` 固定地址放置对 RAM 布局要求很高。

建议实验：

- 先只读懂流程，不急着改 IAP。
- 画出 Flash 分区图和 RAM 分区图。
- 确认 Boot 和 APP 分别的链接地址。

检查点：

- 能说明为什么跳转 APP 前要关闭中断和外设。
- 能说明 CRC32 校验放在写 Flash 前的意义。

## 第 8 阶段：工程整理和可靠性提升

目标：把项目从“能跑”提升到“更好维护”。

建议改进：

- 建立 Git 仓库。
- 添加 `.gitignore`，排除 `OBJ`、`.uvguix.*`、临时文件和编译产物。
- 统一源码编码，优先使用 UTF-8 或明确使用 GBK。
- 把项目说明、引脚表、任务表、分区表写成文档。
- 开启 FreeRTOS 栈溢出检测。
- 检查所有 `xTaskCreate` 的返回值。
- 减少运行期浮点 `printf/sprintf`。
- 对共享状态变量统一加锁或复制快照。

建议输出文档：

- `docs/工程结构.md`
- `docs/引脚分配.md`
- `docs/任务说明.md`
- `docs/IAP分区说明.md`
- `docs/调试记录.md`

## 推荐学习节奏

如果每天学习 1 到 2 小时，可以按下面节奏推进：

- 第 1 周：跑通工程，理解主流程和文件结构。
- 第 2 周：学习 FreeRTOS 任务、优先级、信号量。
- 第 3 周：读懂模式切换、按键、LCD、传感器。
- 第 4 周：重点学习电机 PWM、编码器测速、PID。
- 第 5 周：学习自动风速算法和防回流逻辑。
- 第 6 周：学习 IAP、DMA、CRC32、Flash 写入。
- 第 7 周：整理文档，修复编码和工程管理问题。
- 第 8 周：独立增加一个小功能，比如串口命令调参或 LCD 菜单。

## 最适合练手的小改动

从容易到困难：

1. LCD 增加温湿度和气体浓度显示。
2. 串口周期打印系统状态。
3. 增加第三个风速档位。
4. 增加按键双击事件。
5. 增加串口命令修改 PID 参数。
6. 增加参数保存到 Flash。
7. 重新设计 IAP 分区并验证升级流程。

## 学习时的注意事项

- 每改一个功能，只改一小处，先编译再上板验证。
- 涉及电机、H 桥和 PWM 时，先低占空比测试。
- 涉及 Flash 擦写和 IAP 时，先确认地址，不要盲目下载。
- 遇到 HardFault，优先检查栈、数组越界、中断优先级和非法地址跳转。
- 记录每次修改前后的现象，养成调试日志习惯。
